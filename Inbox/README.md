# Inbox

## Purpose

The Inbox is your **single capture point** for all incoming ideas, tasks, emails, and information that needs processing.

## Philosophy

"Capture everything, process regularly, organize intentionally."

The Inbox follows the GTD (Getting Things Done) principle: capture first, organize later.

## Files

### quick-capture.md
**Purpose**: Lightning-fast captures when you need to record something immediately.

- One line per item
- No formatting required
- Process daily

**When to use**: During meetings, while reading, when inspiration strikes, or anytime you can't fully process an idea right now.

### inbox.md
**Purpose**: Main inbox for more detailed captures that need a few sentences.

- Brief descriptions
- Context and source
- Links to references
- Process daily or every few days

**When to use**: When you have a moment to add context but don't want to fully organize yet.

### to-process.md
**Purpose**: Items that have been initially captured but need categorization.

- Moved from quick-capture.md or inbox.md
- Needs decision on where it belongs
- Process during weekly review

**When to use**: During daily processing, move items here that need deeper thought about categorization.

## Processing Workflow

### Daily Processing (5-10 minutes)

1. **Review quick-capture.md**
   - Move items to inbox.md with added context
   - Or move directly to appropriate destination if obvious

2. **Review inbox.md**
   - Decide destination for each item:
     - **Project**: Create/update project file
     - **Area**: Add to appropriate Area file
     - **Resource**: Save to Knowledge Base or Resources
     - **Task**: Create task file or add to project tasks
     - **Archive**: If no longer relevant
     - **to-process.md**: If needs more thought

3. **Clear processed items**
   - Delete from Inbox once categorized
   - Keep Inbox empty or near-empty

### Weekly Review (Sundays, 30 minutes)

1. **Process to-process.md completely**
   - Make decisions on all items
   - Move to appropriate locations
   - Clear the file

2. **Ensure Inbox is empty**
   - Nothing left uncategorized
   - Ready for new week

3. **Review capture system**
   - Is it working?
   - Do you need adjustments?

## Capture Guidelines

### Do Capture:
- ✅ Tasks and action items
- ✅ Ideas and insights
- ✅ Things to research or learn
- ✅ Links and references
- ✅ Questions to answer
- ✅ Things people tell you
- ✅ Meeting notes (before full processing)
- ✅ Email action items

### Don't Let These Slip:
- ❌ "I'll remember this" - No, you won't. Capture it.
- ❌ "It's too small to write down" - Still capture it.
- ❌ "I'll process it now" - If you can't do it in 2 minutes, capture it.

## Capture Examples

### Good Quick Captures
```
- Research Zettelkasten method for knowledge base
- Call dentist for appointment
- Idea: morning routine automation
- Follow up with Sarah about project deadline
- Book recommendation from John: Atomic Habits
```

### Good Inbox Entries
```
## Task: Set up backup automation
Context: Discussed in team meeting today. Need automated daily backups of personal-assistant folder to cloud storage. Dave recommended rclone.
Source: Team meeting 2026-03-11
Priority: Medium

## Idea: Monthly financial dashboard
Create a visual dashboard showing budget vs actual, savings rate, goal progress. Could use Excel or build custom tool. Would help with monthly reviews.
Source: Personal reflection
Related: Areas/Finance/goals.md
```

## Integration with Email/Calendar

### From Gmail Integration
Emails labeled "assistant" or "tasks" automatically appear in quick-capture.md. Process these during daily review.

### From Calendar
Calendar events sync to Schedule/daily/*.md automatically, no inbox processing needed.

## Tips for Success

1. **Always have access** - Keep quick-capture.md open or use mobile capture app
2. **Capture immediately** - Don't trust memory
3. **Process daily** - Prevent buildup
4. **Weekly deep clean** - Empty everything completely
5. **Keep it simple** - Don't overthink during capture
6. **Trust the system** - You will review it, so just capture and move on

## Red Flags

- **Inbox piling up**: Process more frequently
- **Avoiding captures**: System too complex, simplify
- **Can't find things**: Review your categorization logic
- **Same items re-appearing**: Incomplete processing, make actual decisions
